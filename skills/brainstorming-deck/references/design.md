# Visual reference

The deck's visual language is lifted from a "TOPIC 1 · KNOW THE OUTPUT" layout — small teal topic label up top, large serif headline below, two-column body with bulb-icon list on the left and a dark-navy panel on the right that uses amber checkmark circles to call out outcomes.

## Why this layout

- **Topic label + serif headline** gives every slide a magazine-feature feel without taking much vertical space. The label sets context; the headline sets the question.
- **Bulb icons on the left** read as "techniques / methods / things to do" — they pair naturally with action-oriented content like *what we're building*.
- **Navy panel with amber checkmarks on the right** reads as *outcomes / payoffs / wins* — visually distinct from the technique list, so the slide composes as *cause + effect* rather than two competing lists.
- The amber accent is the only warm colour in the palette — it's reserved for moments that should grab attention (checkmarks, the picked-option pill, the warning icons on the trade-offs slide). Using amber sparingly is what makes it pop.

## Palette

| Token | Hex | Usage |
|---|---|---|
| `navy` | `#1A2761` | Headlines, accent rail, dark-panel background, picked-option card. |
| `teal` | `#0F9FA4` | Topic labels, category labels. Cooler than navy; signals "meta-frame". |
| `amber` | `#E89B2C` | Checkmark circles, picked-option pill, warning triangles. The single attention-grabber. |
| `cream` | `#FAF9F6` | Slide background. Warmer than pure white, easier on the eye. |
| `ink` | `#1A2761` | Primary text on light backgrounds. Same hex as navy — keeps the eye moving along one colour line. |
| `body` | `#5B6470` | Secondary copy on light backgrounds. Muted enough not to compete with `ink`. |
| `rule` | `#D8DCE3` | Card borders, fine dividers. |
| `white` | `#FFFFFF` | Text on dark panels. |

## Typography

- **Headlines**: `Georgia` (universal fallback). Upgrade to `Playfair Display` if you're targeting a specific machine that has it installed.
- **Labels / body**: `Calibri` (universal fallback). Upgrade to `Inter` if you can guarantee it.
- The headline font is the one place to spend personality; everything else is straight-edge sans for legibility.

## Slide-by-slide

| Slide | What it does visually |
|---|---|
| 1. Title | Cream background, thin navy rail on the left for vertical rhythm, teal label, huge serif title, short subtitle separated by a teal underline. |
| 2. Why | Topic label + serif head + single-column list of items with small navy bullets. No icons — these are problems, not techniques. The lack of imagery signals "here's the bad news". |
| 3. Approach + Payoffs | The reference layout. Left: teal "WHAT WE'RE BUILDING" label + bulb-icon list. Right: navy panel with amber "WHY IT SOLVES OUR PROBLEMS" label, white serif headline, four amber-circle checkmark items. |
| 4. Options | Three horizontal cards. The picked one is solid navy with an amber top stripe and amber "PICKED" pill — visually a single beat of warm colour against two cooler rejected cards. |
| 5. Trade-offs | Topic label + serif head + amber warning triangles down the left. The amber returns one more time to signal "pay attention". |
